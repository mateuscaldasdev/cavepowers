# Brainstorming (CavePowers)

Turn ideas into designs via dialogue. No code until design approved.

## HARD GATE

No code. No scaffold. No implementation. Until design approved. Every project. No exceptions.

## Steps (in order)

1. Explore project — files, docs, recent commits
2. Ask clarifying questions — one at a time, multiple choice preferred
3. Propose 2-3 approaches — trade-offs + recommendation
4. Present design — section by section, get approval each section
5. Write spec → `docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md`
6. Self-review spec — no TBDs, no contradictions, no ambiguity
7. User reviews spec — wait for approval
8. Invoke writing-plans skill

## Rules

- One question per message
- Multiple choice > open-ended
- YAGNI — remove unnecessary features
- Present design section by section, not all at once
- Lead with recommended approach

## Spec Self-Review Checklist

- No "TBD", "TODO", "implement later"
- No contradictions between sections
- Focused enough for single plan
- No requirement interpretable two ways

## Output Rules (CavePowers active)

- Drop filler
- Use fragments
- Code/paths/commands exact
- Iron law: no code before design approved
